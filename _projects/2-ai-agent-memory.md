## Overview

Investigated memory mechanisms for AI agents in long-horizon, multi-turn interactions, focusing on context loss, redundant historical information, memory evolution, and response consistency.

## Challenge

- Long conversations can exceed context limits and dilute useful historical information.
- Agent memory systems must balance retrieval quality, storage cost, compression loss, forgetting, and response consistency.
- Memory maintenance interacts with model inference, communication latency, and task scheduling in cloud-edge systems.

## My Contributions

- Surveyed the evolution of agent memory systems from retrieval-augmented memory and hierarchical storage to self-evolving memory.
- Analyzed memory extraction, compression, forgetting, consolidation, and dynamic updating mechanisms.
- Reproduced and analyzed the Mem-alpha memory architecture.
- Formulated memory selection, compression, and updating as a policy-optimization problem.
- Investigated the role of reinforcement learning in long-term memory maintenance.
- Explored memory-aware task scheduling under a cloud-edge collaborative architecture.
- Analyzed tradeoffs among model inference, memory storage, retrieval overhead, communication latency, and response quality.

## Technical Highlights

- Long-horizon memory pipeline analysis
- Retrieval-augmented and hierarchical memory mechanisms
- Policy optimization for memory maintenance
- Cloud-edge AI service scheduling perspective

## Outcomes

- Established a structured understanding of long-horizon agent memory pipelines.
- Built an experimental basis for reinforcement-learning-driven memory maintenance.
- Connected agent memory management with cloud-edge AI service orchestration.
